# PyTorch BCEWithLogitsLoss 실습
본 실습은 모델의 예측 확률 계산(sigmoid)과 비용 함수 계산(BCELoss)을 하나의 모듈로 통합하여, 코드의 간결성과 수치 안정성(Numerical Stability)을 동시에 확보하는 torch.nn.BCEWithLogitsLoss의 사용법을 학습합니다.

## 🛠 주요 학습 내용
 1. BCEWithLogitsLoss 도입: 별도로 torch.sigmoid를 적용할 필요 없이, 선형 계산값(H)을 손실 함수에 직접 전달하여 비용을 계산합니다.

 2. 수치 안정성(Numerical Stability): sigmoid를 거친 후 log를 취하는 과정에서 발생할 수 있는 0이나 1 근처의 계산 오류(로그의 0값 문제)를 방지하는 PyTorch 내부 최적화 메커니즘을 이해합니다.

 3. 모델 출력 구조: 모델의 forward 메서드가 확률값이 아닌 선형 계산값(H)을 반환하도록 설계하고, 예측 시에만 필요에 따라 sigmoid를 적용하는 실무적인 패턴을 학습합니다.

## 🚀 학습 순서
  1. 모델 클래스 정의: nn.Module을 상속받아 선형 레이어만 포함한 Perceptron 모델을 구성합니다.

  2. 손실 함수 설정: nn.BCEWithLogitsLoss를 사용하여 모델의 출력값과 정답 간의 손실을 평가합니다.

  3. 학습 루프 설계: 모델로부터 선형 계산값(H)을 얻습니다.
                    
                    criterion(H, y)를 통해 한 번에 비용을 계산합니다.

                    backward()를 수행하여 자동 미분을 진행합니다.

  4. 예측 단계: 학습된 모델의 출력값(H)에 torch.sigmoid를 적용하여 0~1 사이의 확률값(z)으로 변환하고 분류 결과를 도출합니다.

## 💡 이번 실습의 핵심 포인트
▶ 통합 연산: sigmoid + BCE 조합보다 연산 효율이 뛰어나며, 코드의 가독성이 향상됩니다.

▶ Logit 기반 모델링: 신경망 학습 시 중간 단계에서 확률(0~1)이 아닌 Logit(선형 계산값)을 다루는 것이 왜 권장되는지 이해할 수 있습니다.

▶ 간결성: 손실 함수 호출 한 줄로 모델의 출력과 정답의 차이를 측정할 수 있습니다.
