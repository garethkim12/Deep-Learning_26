# PyTorch nn.Module 기반 퍼셉트론 클래스 실습
본 실습은 이전의 절차적 코드 방식에서 벗어나, PyTorch의 nn.Module 클래스를 상속받아 모델을 직접 정의하고 설계하는 객체 지향적(Object-Oriented) 방식을 익히는 데 목적이 있습니다. 실제 현업에서 사용하는 모델 구조화 방식을 배울 수 있는 단계입니다.

## 🛠 주요 학습 내용
  1. nn.Module 클래스 구조화: __init__ 함수로 모델의 레이어를 정의하고, forward 함수로 데이터가 흐르는 경로를 직접 설계합니다.

  2. 사용자 정의 모델 클래스: class Perceptron(torch.nn.Module):과 같이 모델을 하나의 클래스로 정의하여, 코드의 재사용성과 관리 
     효율성을 높입니다.

  3. 캡슐화된 모델 학습: 모델 자체를 인스턴스로 생성하여 학습 루프 내에서 간결하게 호출합니다.

  4. 표준 학습 파이프라인: nn.Module로 정의된 모델을 활용하여 이진 분류를 수행하는 표준적인 파이토치 학습 흐름을 확립합니다.

## 🚀 학습 순서
    1. 모델 클래스 정의: nn.Module을 상속받아 __init__에서 Linear 레이어를 선언하고, forward에서 연산 순서를 지정합니다.

    2. 모델 인스턴스 생성: 정의한 Perceptron 클래스를 기반으로 실제 모델 객체를 생성합니다.

    3. 학습 준비: criterion (손실 함수)과 optimizer를 모델의 파라미터(model.parameters())와 함께 설정합니다.

    4. 학습 루프 실행: optimizer.zero_grad(), model.forward(), backward(), step()으로 이어지는 객체 지향 학습을 수행합니다.

    5. 예측 결과 확인: 모델을 사용해 새로운 입력 데이터에 대한 농구선수 판별 확률을 계산합니다.

## 💡 이번 실습의 핵심 포인트
▶ 코드의 모듈화: 모델 로직을 별도의 클래스로 분리함으로써, 복잡한 신경망 구조를 체계적으로 관리할 수 있습니다.

▶ 가독성 향상: forward 메서드를 통해 모델이 데이터를 어떤 순서로 처리하는지 직관적으로 파악할 수 있습니다.

▶ 확장성: 추후 더 복잡한 레이어를 추가할 때 모델 클래스만 수정하면 되므로 확장성이 뛰어납니다.
